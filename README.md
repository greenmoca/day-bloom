# DAY BLOOM 🌸

> 나만의 하루를 피워내는 공간

목표·하루 기록·할 일·메모·비전보드·날씨를 한 화면에서 관리하는 개인 다이어리 웹앱입니다. 보라색 테마의 차분한 카드형 대시보드로, 별도 설치 없이 브라우저에서 바로 쓸 수 있고 PWA로 설치도 가능합니다.

---

## 주요 기능

- **메인 대시보드** — 오늘 날짜, 매일 바뀌는 명언(366개, 1년 무중복), 모든 카드를 한눈에
- **목표(Goal)** — 추가/수정/삭제, 완료 체크, 순서 변경, 기간(올해·이번 달·이번 주·직접 설정 날짜 범위)·중요도
- **빠른 추가(Quick Add)** — 기록·메모·할 일을 버튼 하나로
- **데일리 기록** — 제목·감정·카테고리·내용·내일 할 일, 전체/오늘/이번 달/카테고리 필터
- **할 일(To-Do)** — 날짜를 지정해 캘린더에 표시, 완료 체크
- **캘린더** — 월간 보기, 날짜별 할 일 표시 + 감정 색 점, 날짜 클릭 시 그날 상세
- **메모** — 추가/수정/삭제, 검색, 고정, 상태(임시·보관·완료)
- **비전보드** — 이미지 업로드(자동 압축), 설명, 대표 이미지
- **날씨** — API 키 없이 위치 기반 실시간 날씨(Open-Meteo + IP), 매시 자동 갱신, 오프라인 시 마지막 날씨 표시
- **설정** — 테마 색 5종, 글씨 크기, 데이터 내보내기/가져오기(전체 교체·합치기)
- **접근성** — 키보드 조작, 포커스 표시·트랩, 스크린리더 라벨, 동작 줄이기 대응
- **첫 방문 온보딩** — 핵심 기능 환영 카드

## 사용법

1. `index.html`(= `smile-diary.html`)을 브라우저로 열면 바로 시작됩니다.
2. 모든 데이터는 **이 브라우저(LocalStorage)** 에만 저장됩니다.
3. 다른 기기/주소로 옮기려면: 설정 → **내보내기**로 백업 → 새 환경에서 **가져오기**.

## 데이터 & 프라이버시

- 기록은 서버 없이 브라우저에만 저장됩니다. 정기적으로 **내보내기**로 백업하세요(7일 이상 미백업 시 안내가 표시됩니다).
- 날씨를 위해 위치/IP가 외부 서비스(Open-Meteo, ipwho.is)로 전송됩니다. 위치 권한 팝업은 자동으로 뜨지 않으며, "📍 정확한 위치"를 누를 때만 요청합니다.

## 기술

- 순수 HTML + CSS + JavaScript 단일 파일, 외부 빌드 없음
- 저장: LocalStorage
- 날씨: Open-Meteo(현재 날씨·대기질), ipwho.is(IP 위치) — 모두 키 불필요
- PWA: `manifest.json` + 서비스 워커(`sw.js`)

## 배포 (GitHub Pages)

1. 새 저장소 생성(Public).
2. 파일 업로드: `index.html`, `manifest.json`, `sw.js`, `day-bloom-icon-180.png`, `day-bloom-icon-192.png`, `day-bloom-icon-512.png`
   - ⚠️ HTML 파일은 반드시 **`index.html`** 이름으로 올릴 것(GitHub Pages가 루트에서 자동 인식).
   - 모든 파일은 **같은 폴더(루트)** 에 둘 것.
3. Settings → Pages → Source: Deploy from a branch → main / (root).
4. 발급된 `https://<사용자명>.github.io/<저장소>/` 로 접속(HTTPS라 PWA·날씨·위치 모두 정상).

## 업데이트 배포 시 주의

앱(특히 정적 파일)을 수정해 다시 배포할 때마다 **`sw.js`의 `CACHE` 버전 숫자**(`day-bloom-vN`)를 하나 올리세요. 그러면 사용자에게 "새 버전" 안내가 뜨고, 동의하면 갱신됩니다. 이 규칙을 지키면 캐시 때문에 옛 화면이 남는 문제를 막을 수 있습니다.
